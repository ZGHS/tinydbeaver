# TinyDB Studio 实现计划

## 1. 项目概述

TinyDB Studio是一个轻量级数据库客户端，专注于三个核心功能：查看数据、修改数据、导出CSV。设计理念是极简，解决传统数据库客户端启动慢、内存高、界面复杂的问题。

## 2. 技术栈

| 层级 | 技术 | 版本 |
|------|------|------|
| 桌面框架 | Electron | ^20.0.0 |
| 前端框架 | React + TypeScript | React ^18.0.0, TypeScript ^4.9.0 |
| 构建工具 | Vite | ^4.0.0 |
| 表格组件 | ag-grid-react | ^29.0.0 |
| SQLite驱动 | better-sqlite3 | ^8.0.0 |
| PostgreSQL驱动 | pg | ^8.8.0 |
| SQL解析 | node-sql-parser | ^4.0.0 |
| 密码存储 | keytar | ^7.9.0 |
| 执行计划图 | 自定义树组件 | - |

## 4. 核心功能实现

### 4.1 连接管理
- 首次启动时显示连接配置弹窗
- 支持SQLite和PostgreSQL连接
- 密码使用keytar加密存储
- 连接信息保存在本地配置文件
- 提供连接下拉切换功能

### 4.2 数据浏览
- 连接成功后自动加载数据库表列表
- 下拉选择表后立即展示数据
- 使用ag-grid-react实现虚拟滚动，支持10万+行数据
- 自动处理不同数据类型的显示

### 4.3 数据编辑
- 双击单元格进入编辑模式
- 编辑完成后自动生成并执行UPDATE语句
- 右键菜单支持删除行功能
- 提供行级操作的确认机制

### 4.4 SQL执行
- 单行输入框支持SQL语句输入
- 智能补全功能：
  - FROM后提示表名
  - 表名.后提示列名
- 执行按钮触发SQL执行
- 支持SELECT、INSERT、UPDATE、DELETE语句

### 4.5 执行计划
- PostgreSQL专用功能
- 解析EXPLAIN结果
- 渲染为树状图展示执行计划
- 显示关键执行指标

### 4.6 CSV导出
- 一键导出当前表格数据
- 支持自定义导出选项
- 自动处理CSV格式和编码

## 5. 界面设计

### 5.1 布局
- 顶部：连接选择器 + 表选择器
- 中部：数据表格（占主要空间）
- 底部：SQL输入框 + 执行按钮 + 导出按钮
- 无侧边栏树、无多标签页、无图标工具栏

### 5.2 交互设计
- 所有操作不超过2次点击
- 即时反馈：选择表后立即加载数据
- 智能补全：SQL输入时提供上下文相关建议
- 简单直观的错误提示

## 6. 实现步骤

### 6.1 项目初始化
1. 初始化Vite + React + TypeScript项目
2. 配置Electron集成
3. 安装核心依赖包
4. 设置项目基础结构

### 6.2 核心模块开发
1. **数据库连接模块**：实现连接管理、密码存储
2. **数据表格模块**：集成ag-grid，实现数据展示和编辑
3. **SQL执行模块**：实现SQL输入、执行和补全
4. **导出模块**：实现CSV导出功能
5. **执行计划模块**：实现PostgreSQL执行计划解析和展示

### 6.3 界面实现
1. 实现主界面布局
2. 开发各个功能组件
3. 实现组件间的状态管理
4. 优化用户交互体验

### 6.4 测试与优化
1. 功能测试：验证所有核心功能
2. 性能测试：测试大数据量下的性能
3. 兼容性测试：确保在不同平台正常运行
4. 优化启动速度和内存使用

## 7. 风险与应对

### 7.1 潜在风险
- 数据库驱动兼容性问题
- 大数据量下的性能问题
- 跨平台兼容性问题
- 密码存储安全性

### 7.2 应对措施
- 使用稳定版本的驱动库
- 实现数据分页和虚拟滚动
- 充分测试不同平台
- 使用keytar进行安全的密码存储
- 实现错误处理和用户友好的提示

## 8. 预期成果

- 一个轻量级、启动快速的数据库客户端
- 支持SQLite和PostgreSQL数据库
- 实现数据浏览、编辑、SQL执行和CSV导出功能

- 良好的性能和用户体验
